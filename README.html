gtml

A Static Site Generator & Templating System

What is gtml?

gtml is an opinionated static site generator which compiles components and routes into static HTML assets. It's centered around HTML generation, making reusable bits of HTML which you can use throughout your entire webpage.

gtml takes in your routes, resolves the components found within, and compiles everything into static HTML.

Installation

# Build from source

go build -o gtml main.go

# Or install globally

go install main.go

Quick Start

# Initialize a new project

gtml init myapp

# Compile the project

gtml compile myapp

# Watch for changes and recompile

gtml compile myapp --watch

Project Structure

gtml expects a very specific project structure:

./myapp
./myapp/components       # Component definitions
./myapp/routes           # Page routes
./myapp/static           # Static assets
./myapp/dist             # Compiled output

These directories form the foundation of a gtml application and are required. Without these directories, gtml will fail to compile.

CLI Commands

init

Initialize a new gtml project at the specified path:

gtml init <PATH> [--force]

  • PATH - The directory path where the project will be created
  • --force - Overwrite existing directory if it exists

Example: gtml init myapp creates a new project with sample components and routes.

compile

Compile routes and components into static HTML:

gtml compile <PATH> [--watch]

  • PATH - The path to the gtml project
  • --watch - Watch for file changes and recompile automatically

Compiled output will be placed in the dist directory.

Components

Basic Component

Components exist within the components directory and contain a single HTML component file:


<header props='title string'>
  <h1>{title}</h1>
</header>

Component Naming Rules

  • Component files must be named in PascalCase (e.g., Header.html, UserCard.html)
  • Subdirectories are allowed (e.g., components/layouts/MainLayout.html)
  • Components are accessed by their filename, not their path
  • Component names must be unique across the entire components directory

Component Structure Rules

  • Each component must have a single root element
  • Components can contain an optional <style> block at the top
  • The props attribute defines the component's dynamic data

Using Components

Components are used as HTML tags with PascalCase names:

<Header title="Welcome" />
<UserCard name="John" age={30} />

Props

Defining Props

The props attribute is formatted as 'PROPNAME PROPTYPE, PROPNAME PROPTYPE':

<div props='name string, age int, isActive boolean'>
  <p>{name} is {age} years old</p>
</div>

Prop Types

String

Can be passed as raw string or expression:

<!-- Both are valid -->
<User name="John" />
<User name={"John"} />

Int

Must be passed as an expression:

<Counter value={42} />
<Counter value={10 + 5} />  <!-- Evaluates to 15 -->

Boolean

Must be passed as an expression:

<Button disabled={true} />
<Button disabled={isEnabled == false} />

Expressions

Expressions are indicated by {} and are evaluated during compilation. They can be used in text content and attribute values.

Text Content

<p>Hello, {name}!</p>
<p>2 + 2 = {2 + 2}</p>

Attribute Values

<img src={imageUrl} />
<button class={isActive ? 'active' : 'inactive'}>Click</button>

Supported Operations

  • Arithmetic: +, -, *, /, %
  • Comparison: ==, !=, <, >, <=, >=
  • Logical: &&, ||
  • String concatenation: "hello" + " " + "world"

Ternary Operator

Conditional rendering using ternary expressions:

{ isLoggedIn ? (
  <p>Welcome back!</p>
) : (
  <p>Please log in.</p>
) }

Slots

Slots allow you to insert content into components at designated places. This is useful for creating reusable layouts.

Defining Slots in Components

Use the <slot name='slotname' /> syntax:

<!-- components/PageLayout.html -->
<html props='title string'>
  <head>
    <title>{title}</title>
  </head>
  <body>
    <nav>Navigation</nav>
    <slot name='content' />
  </body>
</html>

Using Slots

Pass content to a named slot using the tag attribute:

<PageLayout title="My Page">
  <slot name='content' tag='main'>
    <h1>Welcome</h1>
    <p>This is the main content.</p>
  </slot>
</PageLayout>

Styling

Components can include CSS styles in a <style> block. Styles are automatically scoped to the component.

Component Styles

<style>
  h1 {
    color: blue;
    font-size: 2rem;
  }
  .card {
    padding: 1rem;
    border-radius: 8px;
  }
</style>

<div class="card">
  <h1>{title}</h1>
</div>

Styles are compiled to a single styles.css file in dist/static/ with unique scope IDs to prevent conflicts.

Routes

Routes are HTML files in the routes directory that define your pages.

Route Naming Rules

  • Route files must be named in kebab-case (e.g., about.html, contact-us.html)
  • index.html is a special case for the home page
  • Subdirectories are supported and mirrored in the output

Route Output

Compiled routes mirror the routes directory structure:

./routes/index.html        → ./dist/index.html
./routes/about.html        → ./dist/about.html
./routes/blog/post.html    → ./dist/blog/post.html

Static Assets

Place static files (images, fonts, additional CSS, etc.) in the static directory. These are copied to dist/static during compilation.

Complete Example

Project Structure

./myapp/
├── components/
│   ├── Header.html
│   └── Card.html
├── routes/
│   └── index.html
└── static/

components/Header.html

<style>
  header {
    background: #333;
    color: white;
    padding: 1rem;
  }
</style>

<header props='title string'>
  <h1>{title}</h1>
</header>

components/Card.html

<style>
  .card {
    border: 1px solid #ddd;
    border-radius: 8px;
    padding: 1.5rem;
  }
</style>

<div props='heading string, content string' class="card">
  <h2>{heading}</h2>
  <p>{content}</p>
</div>

routes/index.html

<Header title="My Website" />

<main>
  <Card heading="Welcome" content="This is my first gtml page!" />
</main>

Compiled Output (dist/index.html)

<html>
  <head>
    <link rel="stylesheet" href="/static/styles.css">
  </head>
  <body>
    <header data-header="">
      <h1 data-header="">My Website</h1>
    </header>

    <main>
      <div data-card="" class="card">
        <h2 data-card="">Welcome</h2>
        <p data-card="">This is my first gtml page!</p>
      </div>
    </main>
  </body>
</html>
