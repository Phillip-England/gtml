Installation

Install gtml using Go:

go install github.com/phillip-england/gtml@latest

Verify the installation:

gtml --help

Project Structure

A gtml project requires the following directory structure:

./myapp
./myapp/components    # Your HTML components
./myapp/routes        # Your page routes
./myapp/dist          # Compiled output
./myapp/static        # Static assets (CSS, JS, images)

Note: These directories are required. Without them, gtml will fail to compile.

Hello World

Let's create a simple Hello World example. First, create a component:

components/Greeting.html

<div props='name string'>
  <h1>Hello, {name}!</h1>
</div>

Then use it in a route:

routes/index.html

<html>
  <body>
    <Greeting name="World" />
  </body>
</html>

Run gtml compile ./myapp to get:

dist/index.html

<html>
  <body>
    <div data-greeting="">
      <h1>Hello, World!</h1>
    </div>
  </body>
</html>

Basic Components

Components are reusable HTML snippets stored in the components directory. Each component is a single HTML file.

components/Card.html

<div props='heading string, subheading string'>
  <h1>{heading}</h1>
  <p>{subheading}</p>
</div>

Components can include styles that get scoped to the component:

<style>
  h1 { color: red; }
</style>

<div props='heading string'>
  <h1>{heading}</h1>
</div>

Props

Props are declared using the props attribute on the outermost element. Each prop has a name and type, separated by commas:

<div props='name string, age int, isAdmin boolean'>
  <p>Name: {name}</p>
  <p>Age: {age}</p>
</div>

Props are rendered using curly braces: {propName}

Prop Types

gtml supports three prop types:

String

Strings can be passed as raw values or expressions:

<Greeting name="Bob" />
<Greeting name={"Bob"} />
<Greeting name={"Bo" + "b"} />

Int

Integers must be passed as expressions:

<AgeDisplay age={25} />
<AgeDisplay age={20 + 5} />

Boolean

Booleans must be passed as expressions:

<AdminPanel isAdmin={true} />
<AdminPanel isAdmin={age > 21} />

Expressions

Expressions are evaluated within {} braces. They can be used in text content and attributes.

In Text Content

<div props='name string, age int'>
  <p>My name is {name}</p>
  <p>My age is {age}</p>
</div>

In Attributes

<div props='extraYears int'>
  <AgeComp age={extraYears + 2} />
</div>

Arithmetic Operations

Expressions support standard math operations (PEMDAS order):

<Calculator result={10 + 5 * 2} />  <!-- 20 -->
<Calculator result={(10 + 5) * 2} /> <!-- 30 -->

Slots

Slots allow you to insert content into specific places within a component.

Define a Slot

Use <slot name='...' /> to define where content goes:

components/Layout.html

<html props='title string'>
  <head><title>{title}</title></head>
  <body>
    <nav>My Site</nav>
    <slot name='content' />
  </body>
</html>

Fill a Slot

Use <slot name='...' tag='...'> to fill it:

<Layout title="My Page">
  <slot name='content' tag='main'>
    <h1>Welcome!</h1>
    <p>This is my page content.</p>
  </slot>
</Layout>

The tag attribute specifies what HTML element wraps the slot content.

Prop Drilling

Pass props down through nested components using expressions:

<!-- components/Inner.html -->
<div props='text string'>
  <p>{text}</p>
</div>

<!-- components/Outer.html -->
<div props='title string'>
  <Inner text={title} />
</div>

The parent's title prop is passed to the child's text prop using expression syntax.

Conditional Rendering

Use ternary operators for conditional rendering:

<div props='isAdmin boolean'>
  { isAdmin ? (
    <p>Welcome, Admin!</p>
  ) : (
    <p>Welcome, User!</p>
  ) }
</div>

Comparison Operators

<div props='age int'>
  { age >= 21 ? (
    <p>You can drink</p>
  ) : (
    <p>You cannot drink</p>
  ) }
</div>

Available operators:

  • == equals
  • != not equals
  • < less than
  • > greater than
  • <= less than or equal
  • >= greater than or equal

Logical Operators

<div props='age int'>
  { age > 21 && age < 25 ? (
    <p>Young adult</p>
  ) : (
    <p>Other age group</p>
  ) }
</div>
  • && - both conditions must be true
  • || - at least one condition must be true

GTML Scripts

GTML provides a simple way to add client-side interactivity to your components using <script type='gtml'> blocks. These scripts are compiled into vanilla JavaScript.

<div class="counter">
  <p>Count: <span id="count">0</span></p>
  <button id="increment">+</button>

  <script type='gtml'>
    var count = 0;
    #increment.onclick(function() {
      count++;
      document.getElementById('count').textContent = count;
    })
  </script>
</div>

Note: GTML scripts are compiled to standard JavaScript. The special syntax (signals, selectors) is transformed at build time.

Signals

Signals are reactive variables that automatically update the DOM when changed. Create a signal using $name = value syntax.

Signal Syntax

// Inside a <script type='gtml'> block:

// Create a signal with initial value
$count = 0

// Create a signal from a prop value
$count = $initialValue

// Update a signal
setSignal('count', getSignal('count').value + 1)

// Read a signal value
var currentValue = getSignal('count').value

Signal Functions

  • $name = value - Create/set a signal (compiles to setSignal('name', value))
  • $name - Access a signal value (compiles to getSignal('name').value)
  • getSignal('name') - Get the signal object
  • setSignal('name', value) - Set a signal value

Using Props as Initial Values

Props can be used as initial signal values. When you use $propName on the right side of an assignment, the prop value is used:

// If component has props='startValue int'
$counter = $startValue  // Signal initialized from prop

Element Selectors

GTML provides shorthand selectors for querying DOM elements:

ID Selector

Use #id to select by ID:

// GTML syntax
#myButton.onclick(function() { ... })

// Compiles to
document.querySelector('#myButton').onclick = function() { ... }

Class Selector

Use .class to select by class:

// GTML syntax
.submit-btn.onclick(function() { ... })

// Compiles to
document.querySelector('.submit-btn').onclick = function() { ... }

Event Handlers

Attach event handlers using the .onevent(function() { }) syntax:

<script type='gtml'>
  // Click handler
  #myButton.onclick(function() {
    alert('Button clicked!');
  })

  // Other events work too
  #myInput.onchange(function() {
    console.log('Input changed');
  })

  #myDiv.onmouseover(function() {
    this.style.background = 'blue';
  })
</script>

The syntax .onclick(function() {}) compiles to .onclick = function() {}.

Signal Rendering

Signals automatically render in the DOM. Use curly brace expression syntax with the signal name in your HTML to display signal values.

How It Works

When you reference a signal in your HTML template using curly braces (just like prop expressions), it gets compiled to a <span> element with a data-gtml-signal-value attribute that automatically updates when the signal changes.

The span's content updates automatically whenever you call setSignal().

Complete Example

Here's the structure of an interactive counter component:

<div props='initialValue int' class='counter'>
  <p class='value'>
    <span data-gtml-signal-value='myCount'></span>
  </p>
  <div class='buttons'>
    <button id='decrement'>-</button>
    <button id='increment'>+</button>
  </div>

  <script type='gtml'>
    $myCount = $initialValue
    #decrement.onclick(function() {
      setSignal('myCount', getSignal('myCount').value - 1)
    })
    #increment.onclick(function() {
      setSignal('myCount', getSignal('myCount').value + 1)
    })
  </script>
</div>

Use the component in a route:

<Counter initialValue='10' />

Try it: See live examples of interactive components on the Interactive Components Demo page.

gtml init

Initialize a new gtml project:

gtml init myapp

This creates the required directory structure with starter components:

./myapp
./myapp/components/BasicButton.html
./myapp/components/GuestLayout.html
./myapp/routes/index.html
./myapp/dist/
./myapp/static/

Force overwrite an existing directory:

gtml init myapp --force

gtml compile

Compile your gtml project to static HTML:

gtml compile ./myapp

This compiles all routes in ./myapp/routes and outputs to ./myapp/dist.

Watch mode for development:

gtml compile ./myapp --watch

This automatically recompiles when any file in the project changes.
