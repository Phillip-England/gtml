Loading States

Handle loading and error states with suspense and fallback elements.

Suspense Element

The suspense attribute marks content to show while data is loading:

<div fetch='GET https://jsonplaceholder.typicode.com/posts' as='posts'>
  <div suspense>
    <p>Loading posts...</p>
  </div>
  <ul>
    <li for='post in posts'>{post.title}</li>
  </ul>
</div>

Example: Loading Spinner

A loading spinner that appears while fetching todos:

Loading todos...
  • {todo.title}

Fallback Element

The fallback attribute marks content to show if the request fails:

<div fetch='GET https://api.example.com/data' as='data'>
  <div suspense>
    <p>Loading...</p>
  </div>
  <div fallback>
    <p>Failed to load data. Please try again.</p>
  </div>
  <ul>
    <li for='item in data'>{item.name}</li>
  </ul>
</div>

Complete Example

Combining suspense, fallback, and data display:

<div fetch='GET https://jsonplaceholder.typicode.com/albums' as='albums'>
  <!-- Loading state -->
  <div suspense class="p-4 bg-blue-50 rounded">
    <p class="text-blue-700">Loading albums...</p>
  </div>

  <!-- Error state -->
  <div fallback class="p-4 bg-red-50 rounded">
    <p class="text-red-700">Failed to load albums.</p>
  </div>

  <!-- Success state -->
  <div class="grid grid-cols-2 gap-4">
    <div for='album in albums' class="p-4 bg-white rounded shadow">
      <p>{album.title}</p>
    </div>
  </div>
</div>

Loading albums...

Failed to load albums. Please try again later.

Album #{album.id}

{album.title}

Attributes Reference

Attribute Behavior Description
suspense Shown during fetch Content displayed while the request is in flight. Removed when data loads.
fallback Shown on error Content displayed if the request fails or returns non-JSON data.

When Fallback Triggers

  • Network errors (no connection, timeout)
  • HTTP error status codes (4xx, 5xx)
  • Response is not valid JSON
  • Server returns an unexpected format
